配置可远程访问docker服务

目标:可以本地远程访问服务端的docker服务

如可在本地的idea的插件里, 来连接在云服务器上的docker,并且任意部署发布

配置可远程访问

  1. 修改文件 修改以下这一行

ExecStart=/usr/bin/dockerd -H unix:///var/run/docker.sock -H tcp://0.0.0.0:2375

  1. 重启 daemon
  2. 重启docker服务

上述步骤可以配置远程访问,但此步骤没有任何验证,安全性极差,需在下一步中配置证书登录

配置安全访问,配置证书

参考官方文档
https://docs.docker.com/engine/security/https/

生成证书

生成证书可参考官网,或直接使用此的脚本,与官网步骤一致
- 注意替换脚本里的三个参数
- ip填写主机公网ip即可

配置访问连接

还是修改之前提到的文件docker服务配置文件
/lib/systemd/system/docker.service

  1. 修改文件 修改以下这一行

    注意最后一行 -H unix:///var/run/docker.sock 是开启本地连接,
    -H tcp://0.0.0.0:2375 是开启端口可访问
    另外如使用阿里云等服务器,需要开放端口访问权限
    开启远程访问存在隐患请参考官方文档https://docs.docker.com/engine/security/#docker-daemon-attack-surface

    总之,保管好证书即可

  2. 重启 daemon
  3. 重启docker服务
本地访问docker服务

此处示例为本地idea安装了连接docker的工具,安装插件不再赘述

  • Engine API URL :格式为:https://你的服务器ip:2375
  • Certificates folder :把生成的证书都个copy下来,的本地目录
    • 其实只需要 ca.pem,cert.pem,key.pem 这几个文件

至此,就可以在本地控制面板愉快的操作远程docker服务的镜像,容器了
参考地址
https://www.cnblogs.com/xiaoqi/p/docker-tls.html
https://docs.docker.com/engine/security/https/
